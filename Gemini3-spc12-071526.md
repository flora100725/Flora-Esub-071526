TFDA 醫療器材智慧審查與補件評估平台 (TFDA AI Regulatory Agentic OS & Dashboard)
系統架構與進階技術建置詳細規格說明書 (Technical Specification & Architecture Blueprint)
壹、 系統願景與全局架構設計 (Vision & System Architecture)
1.1 系統定位與商務痛點
在醫療器材查驗登記（Medical Device Registration）的實務流程中，尤以**超音波診斷儀及其搭配之多通道探頭（Transducer Probes）**的審查最為繁複。一款常規的超音波主機往往需要同時申報 15 至 22 款不同頻率範圍與掃描型態（如凸陣 Linear、線陣 Convex、相控陣 Phased Array、腔內 Endocavity）的探頭。
每一款探頭皆需獨立提交生物相容性（ISO 10993 系列）、聲學輸出量測報告（IEC 60601-2-37、IEC 62359）、電磁相容性（EMC, IEC 60601-1-2）以及出廠品質證明等技術文件。當台灣衛生福利部食品藥物管理署（TFDA）審查官針對其中部分探頭指出技術缺失（Technical Deficiencies）並發出補件通知書（Request for Additional Information, RAI）時，法規專員（Regulatory Affairs, RA）與研發小組往往陷入長達數月的報告重組、跨部門協調、以及與國外原廠技術代表反覆溝通的泥淖，嚴重遲滯取證上市時程。
本平台（TFDA AI Regulatory Agentic OS & Dashboard）旨在提供一套高度智慧化、模態協同且具備實時互動的反饋系統。平台結合 Google Agentic Development Kit (ADK) 核心、大語言模型推理矩陣、微型 RAG 安全沙箱，以及高表現力的動態可視化技術。其目的在於：
自動解構補件公文與技術文件，快速歸納缺失條款並建立與 22 款超音波探頭的拓撲映射關聯。
藉由自主代理工作流（Agentic Workflows）自動調用 Google Search 與 Code Execution 工具，針對合規短板自動補齊背景法規知識，並自動執行性能量測數據不確定度之數學計算與確效。
提供完全離線自包含、高保真且符合 "Sleek Interface" 視覺規範的雙工作區交互平台，使法規專員能以一鍵套用範本、語音聽寫、智能微調與多模型比對等高效率交互方式，於數小時內產出合規、語意精準且語調得體之補件官方回覆函（Response Letter）與原廠協調英文郵件。

2.2 Google Search (google_search) 整合規格
當代理程式解析 skill.md 內預定義之評估條款（如高頻線陣探頭之生物相容性 ISO 10993）時，若發現送審材料未明確指定受試型號或試驗標準版本：
觸發條件：條款解析引擎偵測到輸入文本包含關鍵字「未檢附最新版測試報告」、「缺少對應 ISO/IEC 標準」或「標準更迭判定」。
檢索構造：Agent 自動構建精確的搜尋語意查詢（Search Query）。例如：
site:fda.gov OR site:tfda.gov.tw "ultrasound transducer" "ISO 10993-5" biocompatibility sensitization cytotoxicity requirements
數據清洗與置入：將搜尋結果中最具權威性的前 3 筆網頁摘要（含 URL、標準名稱、法規生效日期）轉換為 Context，注入 RAG 模型提示中，防止模型產生幻覺（Hallucination）。
2.3 內置代碼執行 (built_in_code_execution) 整合規格
為落實本系統之「無模擬真實校驗（Anti-AI-Slop）」原則，當使用者上傳包含探頭聲學強度量測數據（如 
 等）之 CSV 或 JSON 檔案時，系統不依靠 LLM 的文本推測來判斷安全性，而是：
呼叫沙箱：調用內部 Python Executor 或 Google Vertex AI Code Interpreter，上傳數據矩陣。
運行算法：在安全的 Python 沙箱內執行聲學功率減衰與機械指數安全極限回歸。

3.2 技能規格 (Skill.md) 生命週期管理
本平台首創「將合規條款視為可編程技能（Compliance as a Programmed Skill）」之概念。在 Agentic OS 中，一項審查技能被完全定義在一個自包含的 skill.md 內：
載入階段（Load）：使用者一鍵加載或貼上特定探頭的 skill.md。平台會實時解析其 frontmatter（技能名稱、核心工具列表、參數限制）與 Markdown 段落（流程描述、系統 System Instructions）。
修改階段（Revise）：RA 專員或系統開發者可在內嵌的 Monaco-style 編輯器中直接修改該技能內容，或於輸入框輸入提示（例如「加入 2026 最新眼科聲學限制與 43°C 腔內熱安全限制要求」），經大語言模型重寫並更新編輯器。
保存與導出（Export）：修改後的技能可以一鍵導出為標準 Markdown 檔，保存回本地系統，建立可重複利用的企業法規技能資產庫。

7.2 解決大文本（Large Scale Text Area）高度重繪造成的 React 幀率驟降問題 (React Performance Tuning)
漏洞診斷：在 Tab 2 (RAI 補件回覆編寫) 中，有 22 款探頭的長文本 textarea。如果使用者在輸入框中打字，每一次 onChange 事件都會觸發 React 根節點狀態（tfdaAuditItems）的全局重新渲染（Re-render）。在低配平板或筆電上，這會造成長達 200ms 的鍵盤延遲，甚至導致瀏覽器失去響應、白屏崩潰。
技術優化：

7.3 PDF Branded 導出完美符合 Bold Typography 列印樣式優化
漏洞診斷：利用瀏覽器原生 window.print() 導出 PDF 時，Tailwind CSS 的 dark: 暗黑主題樣式、滾動容器 overflow-hidden 以及 Flex 佈局會導致導出的 PDF 頁面被攔截截斷（Clipping），且背景變成一片漆黑。
